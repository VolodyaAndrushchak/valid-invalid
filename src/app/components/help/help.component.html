Help

In the url field you should put a valid http/https url which will be tested. You should't put / and ? symbols in the end of your request.

Than, choose http method.

When you did it, you can fill body for POST/PUT and query params for GET/DELETE methods. This data must be in JSON format.

If you have filled the form you can click Start tests button.

Script works with properties of your body or query params. Script tries to set diferent wrong values for every property. Script sets next values in every property: 

{{properties}}

Than, script uses generated cases(body or query params) in separated http requests. In blocks below you can see test case(body or query params), changed property and latency of request.

Pay attention, that currently script works ONLY with not nested objects!

You can click Only main query params or body and script will run only original body or query params in the http request.

However, you can put headers for you request. Key and value must be valid as http headers. Also, you must delete unused empty header/value key pair.
